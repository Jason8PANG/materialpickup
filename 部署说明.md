# 部署说明

## 环境要求

| 项目 | 版本/要求 |
|------|-----------|
| 操作系统 | CentOS 7+ / RHEL 8+ / Ubuntu 20.04+ / Windows Server 2016+ |
| Python | 3.9 及以上 |
| MySQL | 5.7+ 或 8.0 |
| Nginx | 1.18+ (生产环境推荐) |
| 内存 | ≥ 2GB (建议 4GB) |
| 磁盘 | ≥ 10GB 可用空间 |

### 网络环境

| 服务 | 地址 | 端口 |
|------|------|------|
| 应用服务器 | 10.0.6.134 | 5000 (内部) / 80 (外部 Nginx) |
| MySQL | 10.0.6.86 | 33306 |
| LDAP (预留) | 10.0.6.43 | 389 |

## 部署步骤

### 1. 安装系统依赖

```bash
# CentOS / RHEL
yum install -y python3 python3-pip python3-devel gcc nginx mysql
python3 -m pip install --upgrade pip

# Ubuntu / Debian
apt update
apt install -y python3 python3-pip python3-venv nginx mysql-server
python3 -m pip install --upgrade pip
```

### 2. 获取代码

```bash
# 创建应用目录
mkdir -p /data/app
cd /data/app

# 克隆代码 (或通过 scp 上传)
git clone <repository-url> material-kanban
# 或使用 scp 上传
# scp -r 本地路径/material-kanban root@10.0.6.134:/data/app/

cd material-kanban
```

### 3. 创建虚拟环境

```bash
python3 -m venv venv
source venv/bin/activate
```

### 4. 安装 Python 依赖

```bash
pip install -r requirements.txt
```

`requirements.txt` 内容：

```
Flask==3.1.0
Flask-Session==0.8.0
PyMySQL==1.1.1
werkzeug==3.1.3
```

如果生产环境需要 LDAP 认证，额外安装：

```bash
pip install python-ldap==3.4.4
```

如果使用 Gunicorn 部署，额外安装：

```bash
pip install gunicorn
```

### 5. 配置环境变量

创建 `.env` 文件或直接在系统环境变量中配置：

```bash
# 应用安全
export SECRET_KEY="your-strong-random-secret-key-here"

# MySQL 数据库
export MYSQL_HOST="10.0.6.86"
export MYSQL_PORT="33306"
export MYSQL_USER="powerbi"
export MYSQL_PASSWORD="your-database-password"
export MYSQL_DB="materialpickup"

# SMTP 邮件服务
export SMTP_SERVER="smtp.company.com"
export SMTP_PORT="587"
export SMTP_USE_TLS="true"
export SMTP_USER="kanban@company.com"
export SMTP_PASSWORD="your-email-password"
export MAIL_FROM="kanban@company.com"
export MAIL_FROM_NAME="物料领取看板系统"

# 系统基础 URL (用于邮件审批链接)
export BASE_URL="http://kanban.company.com"
```

> 也可以直接编辑 `app/config.py`，但建议通过环境变量配置以便环境隔离。

### 6. 创建数据库和用户

```sql
-- 登录 MySQL
mysql -h 10.0.6.86 -P 33306 -u root -p

-- 创建数据库
CREATE DATABASE IF NOT EXISTS materialpickup 
  DEFAULT CHARACTER SET utf8mb4 
  DEFAULT COLLATE utf8mb4_unicode_ci;

-- 创建用户并授权（如果使用独立用户）
CREATE USER IF NOT EXISTS 'powerbi'@'10.0.6.134' IDENTIFIED BY 'your-password';
GRANT ALL PRIVILEGES ON materialpickup.* TO 'powerbi'@'10.0.6.134';
FLUSH PRIVILEGES;
```

### 7. 初始化数据库表

```bash
# 确保环境变量已配置
export MYSQL_PASSWORD="your-database-password"
source venv/bin/activate

# 执行初始化
python -m app.init_db
```

执行成功后应看到输出：

```
开始创建数据库表...
  [OK] 表 kr_material_request 已创建/已存在
  [OK] 表 kr_role_mapping 已创建/已存在
  [OK] 表 kr_approval_token 已创建/已存在
  [OK] 表 kr_operation_log 已创建/已存在
开始插入初始数据...
初始数据插入完成!
数据库初始化完成!
```

### 8. 验证部署

```bash
# 启动开发服务器验证
source venv/bin/activate
python run.py

# 在浏览器中访问 http://10.0.6.134:5000
# 使用初始账号登录测试
```

### 9. 生产环境启动 (Gunicorn)

```bash
# 创建 systemd 服务

cat > /etc/systemd/system/material-kanban.service << 'EOF'
[Unit]
Description=Material Requisition Kanban System
After=network.target

[Service]
User=www-data
Group=www-data
WorkingDirectory=/data/app/material-kanban
EnvironmentFile=/data/app/material-kanban/.env
ExecStart=/data/app/material-kanban/venv/bin/gunicorn \
    -w 4 \
    -b 127.0.0.1:5000 \
    --timeout 120 \
    --access-logfile /var/log/material-kanban/access.log \
    --error-logfile /var/log/material-kanban/error.log \
    run:app
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
EOF

# 创建日志目录
mkdir -p /var/log/material-kanban
chown -R www-data:www-data /var/log/material-kanban

# 重新加载并启动
systemctl daemon-reload
systemctl enable material-kanban
systemctl start material-kanban

# 查看状态
systemctl status material-kanban
```

## Nginx 反向代理配置

### 基础配置

```nginx
upstream kanban_backend {
    server 127.0.0.1:5000;
    # 如果有多实例，可添加更多
    # server 127.0.0.1:5001;
}

server {
    listen 80;
    server_name kanban.company.com 10.0.6.134;

    # 如果配置 HTTPS，301 跳转
    # return 301 https://$host$request_uri;

    # 访问日志
    access_log /var/log/nginx/kanban-access.log;
    error_log  /var/log/nginx/kanban-error.log;

    # 请求体大小限制 (防止大申请单提交失败)
    client_max_body_size 10m;

    location / {
        proxy_pass http://kanban_backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # WebSocket 支持 (预留)
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }

    # Session 刷新避免 Nginx 缓存
    location ~* \.(css|js|png|jpg|gif|ico)$ {
        expires 7d;
        add_header Cache-Control "public, immutable";
    }
}
```

### HTTPS 配置 (推荐)

```nginx
server {
    listen 443 ssl http2;
    server_name kanban.company.com;

    ssl_certificate     /etc/nginx/ssl/kanban.pem;
    ssl_certificate_key /etc/nginx/ssl/kanban.key;
    ssl_protocols       TLSv1.2 TLSv1.3;
    ssl_ciphers         HIGH:!aNULL:!MD5;

    # 其余配置同基础配置
    ...
}
```

### 配置生效

```bash
# 测试配置
nginx -t

# 重新加载
nginx -s reload
```

## 数据库初始化 (`init_db.py`) 说明

### 脚本位置

```bash
# 项目根目录下
python -m app.init_db
```

### 功能

- 创建 4 张核心业务表 (`IF NOT EXISTS`，幂等执行)
- 插入 6 条初始角色映射数据

### 创建的数据库表

| 表名 | 用途 |
|------|------|
| `kr_material_request` | 领料申请单主表，存储所有申请及流转信息 |
| `kr_role_mapping` | 域账号-角色映射表，控制登录权限 |
| `kr_approval_token` | 邮件审批一次性令牌表，用于邮件链接审批 |
| `kr_operation_log` | 操作日志审计表 |

### 初始种子数据

| 域账号 | 显示名 | 角色 | 邮箱 |
|--------|--------|------|------|
| admin | 管理员 | admin | admin@example.com |
| supervisor1 | 主管张三 | supervisor | supervisor1@example.com |
| worker1 | 领料员李四 | requester | worker1@example.com |
| warehouse1 | 仓管王五 | warehouse | warehouse1@example.com |
| worker2 | 领料员赵六 | requester | worker2@example.com |
| warehouse2 | 仓管陈七 | warehouse | warehouse2@example.com |

> 初始密码 = 域账号。如需修改密码，在 `kr_role_mapping` 表的 `password_hash` 字段中写入 werkzeug 生成的密码哈希。

### 使用注意事项

- **幂等性**：表使用 `CREATE TABLE IF NOT EXISTS`，重复执行不会报错
- **种子数据**：使用 `INSERT IGNORE`，重复执行不会重复插入
- **执行前请确认**：`MYSQL_PASSWORD` 环境变量已正确设置
- **手动建表**：也可直接执行 `sql/init.sql` 中的 SQL 脚本

### 备份数据库

```bash
#!/bin/bash
# 备份脚本示例: /usr/local/bin/backup-kanban.sh

BACKUP_DIR="/data/backup/material-kanban"
DATE=$(date +%Y%m%d_%H%M%S)
DB_HOST="10.0.6.86"
DB_PORT="33306"
DB_USER="powerbi"
DB_PASS="your-database-password"
DB_NAME="materialpickup"

mkdir -p $BACKUP_DIR

# 使用 mysqldump 备份
mysqldump -h $DB_HOST -P $DB_PORT -u $DB_USER -p$DB_PASS \
  --single-transaction --routines --triggers \
  $DB_NAME > $BACKUP_DIR/${DB_NAME}_${DATE}.sql

# 压缩
gzip $BACKUP_DIR/${DB_NAME}_${DATE}.sql

# 保留最近 30 天
find $BACKUP_DIR -name "*.sql.gz" -mtime +30 -delete

echo "数据库备份完成: ${DB_NAME}_${DATE}.sql.gz"
```

设置定时任务：

```bash
crontab -e

# 每天凌晨 2 点备份
0 2 * * * /usr/local/bin/backup-kanban.sh
```

### 性能调优建议

1. **Gunicorn worker 数**：`-w $(nproc)`，即 CPU 核心数的 2-4 倍
2. **Session 存储**：生产环境建议从 `filesystem` 切换为 `redis`
3. **MySQL 连接池**：当前使用短连接，高并发场景可考虑添加连接池
4. **静态资源**：Nginx 直接托管 `static/` 目录，绕过 Flask

### 故障排查

**问题：登录提示 "密码错误"**
- 检查 `kr_role_mapping` 表中用户是否存在且 `is_active = 1`
- 若 `password_hash` 为空，使用域账号作为密码

**问题：邮件发送失败**
- 检查 `SMTP_*` 环境变量是否正确
- 检查 SMTP 服务器网络可达性
- 查看日志 `journalctl -u material-kanban` 中的邮件错误信息

**问题：看板页面空白或 500 错误**
- 检查 MySQL 连接是否正常
- 检查表结构是否完整（可重新执行 `python -m app.init_db`）
- 查看错误日志 `/var/log/material-kanban/error.log`

**问题：Session 频繁过期**
- 检查 `PERMANENT_SESSION_LIFETIME` 配置（默认 8 小时）
- 检查 `filesystem` Session 存储目录的磁盘空间
